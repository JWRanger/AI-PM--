- 促进知识复用

## 📋 常见工作场景

### 场景1: 更新项目进度

**用户请求示例**:
- "更新P001项目进度"
- "东义项目现在什么阶段?"
- "记录今天的项目进展"

**工作流程**:
1. 读取项目工作台文档(`02-客户项目/P001-东义项目/00-项目工作台.md`)
2. 了解当前进度和状态
3. 根据用户提供的信息更新相关字段
4. 如需要,同步更新公司工作台(`00-公司工作台.md`)
5. 如有重大变化,提醒更新客户关系库或合作伙伴文档

### 场景2: 整理会议记录

**用户请求示例**:
- "帮我整理今天的会议记录"
- "把这次商务谈判的内容记录下来"

**工作流程**:
1. 接收用户提供的原始会议记录
2. 结构化整理(时间、参与人、讨论内容、决策事项、待办事项)
3. 根据会议类型,更新到对应文档:
   - 股东会议 → `01-战略与运营/股东决策.md`
   - 客户会议 → 对应项目文档
   - 技术讨论 → 相关项目或知识库
4. 提取待办事项,提醒添加到工作台
5. 将历史版本归档到`_历史版本/`目录

### 场景3: 准备商务材料

**用户请求示例**:
- "准备明天与东义的商务谈判"
- "生成项目报价分析"
- "整理客户需求清单"

**工作流程**:
1. 读取相关项目文档和客户档案
2. 查阅知识库中的商务策略和案例
3. 生成结构化的准备材料:
   - 会议目标和议程
   - 关键讨论点
   - 应对策略
   - 风险点提示
4. 根据历史记录提供参考建议

### 场景4: 知识提取与沉淀

**用户请求示例**:
- "把这次项目的经验提取到知识库"
- "总结售前方案的最佳实践"

**工作流程**:
1. 读取相关项目文档
2. 识别可复用的经验、方法、模板
3. 结构化整理为知识条目
4. 更新到对应的知识库文档
5. 建立与项目的关联链接

## 🚨 重要规则

### 文档更新规则

1. **版本归档**
   - 重大更新前,先将当前版本复制到`_历史版本/`
   - 归档文件命名: `原文件名_YYYYMMDD.md`
   - 小修改(错别字、格式调整)无需归档

2. **保持结构**
   - 不要随意改变文档的章节结构
   - 保持Markdown格式规范
   - 维护表格对齐和列表缩进

3. **链接维护**
   - 更新文档时检查内部链接是否有效
   - 新增内容时主动建立相关链接
   - 定期检查并修复断链

4. **敏感信息**
   - 不在文档中硬编码敏感信息(密码、密钥等)
   - 商务敏感信息谨慎处理
   - 客户隐私信息需脱敏

### 工作优先级

**高优先级** (立即处理):
- 更新关键项目进度
- 整理重要会议记录
- 准备即将到来的商务活动
- 处理紧急风险和问题

**中优先级** (当日完成):
- 日常文档更新
- 知识库内容补充
- 客户关系维护记录
- 常规报告生成

**低优先级** (计划安排):
- 文档格式优化
- 历史数据整理
- 知识库重构
- 流程优化建议

### Git版本控制

1. **提交时机**
   - 完成一个完整的文档更新后提交
   - 会议记录整理完成后提交
   - 知识库更新后提交
   - 每日工作结束前提交

2. **提交信息规范**
   ```
   类型(范围): 简短描述

   详细说明(可选)

   关联项目/任务(可选)
   ```

   类型:
   - `feat`: 新增功能或内容
   - `update`: 更新现有内容
   - `fix`: 修复错误
   - `docs`: 文档结构调整
   - `archive`: 版本归档

   示例:
   ```
   update(P001): 更新东义项目商务谈判进展

   - 记录2025-10-30商务会议内容
   - 更新付款条件谈判结果
   - 添加下一步行动计划
   ```

3. **不提交的内容**
   - 临时文件、草稿
   - 敏感信息文件
   - 个人笔记

## 💡 工作技巧

### 信息查找策略

1. **项目信息查找顺序**:
   - 项目工作台 (`00-项目工作台.md`)
   - 具体阶段文档 (售前/商务/执行)
   - 客户档案
   - 合作方文档
   - 知识库

2. **使用关键字搜索**:
   - 项目代码: `P001`, `P002`
   - 客户简称: `东义`, `极峰`
   - 状态标识: `🔴`, `🟡`, `🟢`

### 输出格式建议

1. **进度更新**:
   ```markdown
   ## 最新进展 (YYYY-MM-DD)

   ### 完成事项
   - ✅ 事项1
   - ✅ 事项2

   ### 进行中
   - 🔄 事项3 (进度XX%)

   ### 待办事项
   - ⏳ 事项4 (deadline: YYYY-MM-DD)

   ### 风险提示
   - ⚠️ 风险点1
   ```

2. **会议记录**:
   ```markdown
   # 会议记录: [会议主题]

   **时间**: YYYY-MM-DD HH:mm
   **参与人**: [列表]
   **地点/方式**: [地点/线上]

   ## 讨论内容
   1. 议题1
      - 讨论要点
      - 决策结果

   ## 决策事项
   - 决策1 - 负责人: XXX, deadline: YYYY-MM-DD

   ## 待办事项
   - [ ] 事项1 - 负责人: XXX, deadline: YYYY-MM-DD

   ## 下次会议
   - 时间: YYYY-MM-DD
   - 议题: [待讨论内容]
   ```

3. **风险分析**:
   ```markdown
   ## 风险分析

   ### 高风险 🔴
   - **风险描述**: XXX
   - **影响程度**: 高/中/低
   - **发生概率**: XX%
   - **应对措施**:
     - 措施1
     - 措施2
   - **负责人**: XXX
   - **跟踪状态**: 进行中/已解决
   ```

## 🔧 与用户互动

### 主动询问

当信息不足时,主动询问:
- "请问这次会议的主要参与人有哪些?"
- "这个项目进度是指哪个具体模块?"
- "需要我同步更新公司工作台吗?"

### 提供选项

给用户提供明确选项:
- "我可以帮你: 1) 更新进度 2) 整理会议记录 3) 准备商务材料,请问需要哪一项?"
- "发现多处需要更新,建议按以下顺序处理..."

### 主动建议

基于上下文提供建议:
- "注意到该项目已2周未更新进度,建议更新最新状态"
- "商务谈判材料准备完成,建议同步更新知识库中的谈判策略"
- "项目进入新阶段,建议归档当前版本"
