# Claude Web 使用指南 - 汇炼智能

> 本文档指导如何在Web版Claude中高效使用汇炼智能项目文档

---

## 🎯 Web版定位

Claude Web主要用于:
- 移动场景快速查询
- 外出时信息检索
- 多端信息同步
- 轻量级信息获取

**不适合**:
- 复杂的文档编辑
- 批量更新操作
- Git版本控制
- 深度分析和报告生成

## 💬 常见查询场景

### 场景1: 快速进度查询

**提问示例**:
- "东义项目现在什么进度?"
- "M1模块进展如何?"
- "本周有哪些重点工作?"

**Claude会做什么**:
1. 检索项目工作台文档
2. 提取最新进度信息
3. 简洁清晰地回答
4. 必要时提示需要关注的风险

**示例对话**:
```
你: "东义项目现在什么进度?"

Claude: "根据项目工作台信息:

当前阶段: 商务洽谈阶段
整体进度: 70%
关键状态:
- ✅ 售前方案已完成并获客户认可
- 🔄 商务条款谈判中(付款条件待定)
- ⏳ 预计11月完成签约

下一步重点:
1. 本周确定付款条件
2. 准备合同草案

风险提示: 付款条件谈判可能影响签约时间"
```

### 场景2: 会议准备

**提问示例**:
- "明天要和东义商务谈判,帮我准备"
- "下周股东会需要准备什么?"
- "准备极峰科技的技术对接会"

**Claude会做什么**:
1. 读取相关项目和客户档案
2. 查阅商务策略知识库
3. 生成会议准备清单
4. 提供关键讨论点和策略建议

**示例对话**:
```
你: "明天要和东义商务谈判,帮我准备"

Claude: "为您准备了商务谈判材料:

## 会议目标
- 确定最终付款条件
- 推进合同签订时间表

## 谈判要点
1. 付款条件(重点)
   - 我方建议: 3:3:3:1 (签约:验收M1:验收M2/M3:质保期)
   - 预期让步空间: 可接受2:3:3:2

2. 交付时间
   - M1: 3个月
   - M2/M3: 各2个月

## 应对策略
[根据商务策略库提供建议]

## 风险提示
- 注意客户对分期付款的接受度
- 准备好极峰科技的技术实力证明

## 需要带的材料
- 合同草案
- 项目实施计划
- 极峰技术方案
```

### 场景3: 回顾历史信息

**提问示例**:
- "上次和东义谈了什么?"
- "售前阶段有哪些关键决策?"
- "之前客户提出过什么顾虑?"

**Claude会做什么**:
1. 检索相关历史记录
2. 提取关键信息
3. 按时间或主题整理
4. 关联当前状态提供参考

### 场景4: 知识库查询

**提问示例**:
- "如何应对客户议价?"
- "煤化工行业的特点是什么?"
- "售前演示有什么技巧?"

**Claude会做什么**:
1. 搜索知识库相关内容
2. 提取最相关的知识条目
3. 结合当前项目给出建议
4. 必要时引用成功案例

## 💡 使用技巧

### 1. 上传时机

**建议上传频率**:
- 日常更新后: 每天1次
- 重大变更后: 立即上传
- 会议前: 上传最新版本
- 出差前: 确保文档最新

### 2. 提问技巧

**清晰具体**:
✅ "东义项目M1模块现在什么进度?"
❌ "项目怎么样了?"

**指明范围**:
✅ "从商务策略库中找应对议价的方法"
❌ "客户砍价怎么办?"

**提供上下文**:
✅ "明天要和东义谈付款条件,客户倾向于后付,我们怎么应对?"
❌ "明天谈判怎么准备?"

### 3. 多文档关联查询

善用文档间的关联:
```
你: "东义项目涉及哪些合作方?他们的职责是什么?"

Claude: [自动关联项目文档和合作伙伴文档]
"东义项目(P001)涉及以下合作方:

1. 极峰科技
   - 角色: 技术开发合作伙伴
   - 职责: AI算法开发、平台实施
   - 分成: 待定(预计30-40%)
   - 详见: [合作方-极峰/协作手册.md]

2. 小穆总(资源方)
   - 角色: 客户资源引荐
   - 贡献: 引荐东义集团客户
   - 详见: [客户关系库/客户总览.md]"
```

### 4. 快速决策支持

需要快速决策时:
```
你: "东义想改付款比例为2:2:4:2,能接受吗?"

Claude: [快速分析]
"基于当前信息分析:

风险评估:
- 前期回款减少(从60%降到40%)
- 后期集中付款风险增加
- 现金流压力较大

建议:
- 可接受,但需附加条件:
  1. M2/M3验收标准明确化
  2. 延期付款违约条款
  3. 质保金可协商降到1%

是否接受取决于:
- 公司当前现金流状况
- 对客户信用的评估
- 竞争压力大小

建议: 与股东讨论后决策"
```

## 🔄 与Claude Code协同

### 工作流建议

**早上 (外出前)**:
1. Claude Code: 查看今日待办
2. Web Claude: 上传最新文档
3. 记住今日关键事项

**白天 (外出时)**:
1. Web Claude: 快速查询
2. 会议准备
3. 决策支持

**晚上 (回到电脑前)**:
1. Claude Code: 更新文档
2. 整理会议记录
3. 提交Git更新
4. 准备明日Web版文档

### 信息同步

```
[本地] Claude Code
   ↓ 编辑、更新、Git
[文档更新]
   ↓ 导出、上传
[云端] Web Claude
   ↓ 查询、分析
[移动使用]
```

## ⚠️ 注意事项

### 隐私安全

1. **敏感信息**
   - 不要上传包含密码、密钥的文档
   - 商务敏感信息谨慎上传
   - 客户隐私需脱敏

2. **会话记录**
   - Web对话会保存在云端
   - 避免在对话中直接输入机密
   - 定期清理不需要的会话

### 数据一致性

1. **定期更新**
   - 重大更新后及时上传
   - 避免使用过期版本
   - 上传前确认是最新版本

2. **版本标识**
   - 文档中标注更新时间
   - 便于判断是否需要重新上传
